Here’s a deep, thoughtful breakdown of every major component, tool, and service used in the **onXmariners** project – both the public portfolio and the private admin dashboard.

---

## 🧠 Overall Project Architecture

The project is split into **three main parts**:
1. **Public Portfolio Website** – showcases services, skills, testimonials, and a contact form.
2. **Admin CRM Dashboard** – private inbox to view messages, reply, delete, mark as read, and view analytics.
3. **Backend Server** – handles API requests, stores data in MongoDB, sends emails, and authenticates the admin.

All three work together to form a complete freelance/agency system.

---

## 🖥️ Frontend Technologies (Portfolio + Dashboard)

### Core Languages
- **HTML5** – structure of all pages.
- **CSS3** – styling, animations, responsive design, dark theme.
- **JavaScript (ES6+)** – interactivity, API calls, dynamic content.

### Libraries & Frameworks
- **Font Awesome 6** – icons (social media, menu, etc.).
- **Google Fonts** – Syne (headings) and DM Sans (body).
- **Chart.js** – for analytics charts (donut & line charts in admin dashboard).
- **Canvas API** – particle background animation on hero section.

### Features & Custom Code
- **Typewriter effect** – cycles through roles (Marine Engineer, Content Creator, Website Builder).
- **Particle background** – floating gold dots with smooth animation.
- **Intersection Observer** – triggers skill bar fill and stat counters when they come into view.
- **Infinite scrolling testimonials** – right-to-left animation with YouTube iframes.
- **Mobile responsive menu** – hamburger toggle with CSS + JS.
- **Contact form** – collects name, email, project type, budget, message; sends to backend.
- **Live YouTube stats** – fetches real subscriber & view counts via YouTube Data API v3.
- **Admin dashboard** – inbox grid, reply/compose modals, delete with confirm, mark as read, export CSV/PDF, real-time auto‑refresh every 10 seconds.
- **JWT authentication** – login overlay, token storage in localStorage, authFetch wrapper.
- **Link attachment** – allows pasting URLs (YouTube, Drive, etc.) when replying, appended to email body.

### Styling Approach
- **CSS Custom Properties (variables)** – gold, backgrounds, surfaces.
- **Glassmorphism / backdrop‑blur** – on cards and modals.
- **Dark theme** – #080808 background, gold accents.
- **Fully responsive** – breakpoints for tablet (1024px), mobile (768px), small mobile (480px).

## ⚙️ Backend Technologies (Node.js + Express)

### Core Stack
- **Node.js** – runtime environment.
- **Express.js** – web framework for API routes.
- **MongoDB** – database for storing contact messages.
- **Mongoose** – ODM for schema definition and data operations.
- **Resend** – email sending service (for replies).
- **JSON Web Token (JWT)** – authentication for admin routes.
- **bcrypt** – password hashing for admin login.
- **dotenv** – environment variable management.
- **CORS** – cross-origin requests (frontend/backend separation).
- **fs (file system)** – used for testimonials JSON (planned) and static file serving.

### API Endpoints
| Method | Endpoint | Purpose | Authentication |
|--------|----------|---------|----------------|
| POST | `/api/contact` | Receive contact form submission | Public |
| POST | `/api/login` | Admin login, returns JWT | Public |
| GET | `/api/messages` | Get all messages | JWT required |
| POST | `/api/reply` | Send reply email via Resend | JWT required |
| PUT | `/api/read/:id` | Mark message as read | JWT required |
| DELETE | `/api/delete/:id` | Delete a message | JWT required |
| POST | `/api/testimonials` | Add testimonial (optional) | JWT required |
| GET | `/api/testimonials` | Fetch testimonials | Public |

### Security Measures
- **JWT middleware** protects all admin routes.
- **bcrypt** stores admin password hash (never plain text).
- **Environment variables** (JWT_SECRET, ADMIN_EMAIL, ADMIN_PASSWORD_HASH, MONGO_URI, RESEND_API_KEY) kept secret on Render.
- **CORS** configured to allow only the frontend origin (or all during development).
- **Input validation** on backend (required fields, email format).

## ☁️ Deployment & Hosting

- **Backend** – [Render.com](https://render.com) (Node.js service).  
  - Environment variables set in Render dashboard.
  - Automatic deploys from GitHub.
- **Frontend (Portfolio + Dashboard)** – hosted on **Firebase Hosting** (from favicon error `onxmariners.web.app`).  
  - Could also be on Netlify, Vercel, or any static host.
- **Database** – MongoDB Atlas (free tier) or similar cloud MongoDB.
- **Version Control** – Git with GitHub (commits mentioned in console logs).

## 🧠 Summary – Why These Choices?

| Requirement | Chosen Solution | Reason |
|-------------|----------------|--------|
| Fast, interactive frontend | Vanilla JS + CSS | No heavy framework, lightweight, full control. |
| Real‑time inbox | Fetch + auto‑refresh | Simple, no WebSocket overhead. |
| Authentication | JWT + bcrypt | Stateless, secure, easy to implement. |
| Database | MongoDB + Mongoose | Flexible schema, cloud‑friendly, free tier. |
| Email sending | Resend | Simple API, good deliverability, free tier. |
| Charts | Chart.js | Lightweight, easy to integrate. |
| Hosting | Render + Firebase | Free tiers, easy deploys, reliable. |
| Version control | Git + GitHub | Industry standard. |
